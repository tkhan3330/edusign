# EduSign Enterprise (v3) — School Lesson Plan Approval System

A powerful, secure, and intuitive digital signing tool designed specifically for educational institutions. EduSign sits natively on top of Google Drive, allowing Academic Heads to securely preview, convert, and digitally stamp lesson plans uploaded by teachers.

## ✨ New in v3.0 (Enterprise Upgrade)

- **Interactive Setup Wizard:** No more messing with `.env` files. Configure Google Drive credentials and master passwords directly from your browser on first boot!
- **Word Document Support:** Automatically converts Microsoft Word (`.docx`, `.doc`) lesson plans into perfectly formatted PDFs before stamping.
- **Secure Authentication:** Built-in session-based authentication with a secure master password.
- **Multi-Folder Architecture:** Support for pulling files from unlimited folder sources across different grades or subjects.
- **Enhanced UI/UX:** A stunning, fully responsive interface with dynamic PDF previews and professional signature stamps.

---

## 🛠 Features

- **Google Drive Integration:** Streams files directly from Drive without relying on local storage.
- **Professional Digital Stamp:** Adds a legally styled "DIGITALLY APPROVED" stamp with the signer's name, title, signature, and localized timestamp.
- **Zero-Friction Workflow:** The signed PDF replaces the original in Drive seamlessly, preserving revision history.
- **File Status Tracking:** Visually tracks which files are signed and approved directly in the dashboard.
- **100% Free to Host:** Architected to run entirely on free-tier cloud providers (Vercel & Railway).

## 🚀 Quick Start (Local Development)

### 1. Prerequisites
- Node.js ≥ 18
- Google Cloud Account (for Drive API)

### 2. Google Cloud Setup (One-time)
1. Go to [Google Cloud Console](https://console.cloud.google.com/).
2. Create a new project and **Enable the Google Drive API**.
3. Create a **Service Account** and grant it the **Editor** role.
4. Download the JSON key and save it as `service-account.json`.

### 3. Installation
Clone the repository and install dependencies:

```bash
git clone https://github.com/tkhan3330/edusign.git
cd edusign

# Install Backend
cd backend
npm install

# Install Frontend
cd ../frontend
npm install
```

### 4. Running the App
Start both servers.

**Backend:**
```bash
cd backend
npm run dev
```

**Frontend:**
```bash
cd frontend
npm start
```

### 5. The Setup Wizard
Open `http://localhost:3000` in your browser. Since this is your first time running EduSign, the **Interactive Setup Wizard** will automatically launch. 
1. Upload your `service-account.json`.
2. Set your Admin Master Password.
3. Link your Google Drive folders.
4. Start signing!

## 🔒 Security & Best Practices

- **Never Commit Secrets:** Your `service-account.json`, `.env`, and `config.json` are heavily restricted and automatically ignored by Git. **Do not** upload them to GitHub.
- **Stateless Architecture:** No databases are required. All state is securely mapped via Google Drive metadata.
- **Encrypted Passwords:** The admin password is cryptographically salted and hashed via Node's native `scrypt`.

## ☁️ Production Deployment

EduSign is designed to be hosted for **free** on Vercel (Frontend) and Railway (Backend).

### Deploy Backend (Railway)
1. Import your GitHub repo into Railway.
2. Set the Root Directory to `backend`.
3. Add environment variables:
   - `NODE_ENV`: `production`
   - `PORT`: `3001`
   - `FRONTEND_URL`: `https://your-vercel-app-url.vercel.app`
4. Deploy!

### Deploy Frontend (Vercel)
1. Import your GitHub repo into Vercel.
2. Set the Root Directory to `frontend`.
3. Add environment variables:
   - `REACT_APP_API_URL`: `https://your-railway-app-url.up.railway.app/api`
4. Deploy!

Once deployed, visit your Vercel URL to complete the Setup Wizard in production.

### Troubleshooting: Setup Wizard Reappears on Production
If you host your backend on a platform like Railway or Render, you may notice that the Setup Wizard reappears after a few days or after a redeployment. This happens because these platforms use **ephemeral (temporary) file systems** by default. When the server goes to sleep or restarts, the `config.json` and `service-account.json` files generated by the Setup Wizard are erased.

To fix this permanently, you have two options:

**Option 1: Attach a Persistent Volume (Recommended)**
1. Go to your backend service settings in Railway.
2. Under the **Volumes** section, click **Add Volume**.
3. Set the Mount Path to `/app`.
4. Wait for the app to redeploy, then complete the Setup Wizard one last time. Your data will now be permanent.

**Option 2: Use Environment Variables (Bypasses Wizard)**
Add these Environment Variables to your backend service so it always boots up configured:
- `GOOGLE_SERVICE_ACCOUNT_JSON`: Paste the entire contents of your Google service account JSON file.
- `PARENT_FOLDER_IDS`: The Google Drive folder ID(s).
- `ADMIN_PASSWORD`: Your desired login password.
